# איך עובדים עם Claude Code + GitHub + Vercel

## הזרימה בקצרה

```
Claude Code עושה שינויים בקבצים
        ↓
   Commit = שמירה מקומית (כמו "שמור" בוורד)
        ↓
   Push = העלאה ל-GitHub (כמו "העלאה לענן")
        ↓
   Merge = מיזוג ל-main (כמו "פרסם לאתר")
        ↓
   Vercel רואה שינוי ב-main → בונה ומעלה לאוויר אוטומטית
```

---

## מה כל מושג אומר

| מושג | בעברית פשוטה | מתי קורה |
|------|-------------|-----------|
| **Commit** | "שמרתי נקודת ציון" - כמו Save | Claude Code עושה את זה אוטומטית |
| **Push** | "העלתי את השמירות ל-GitHub" | Claude Code עושה את זה אוטומטית |
| **Branch** | "עותק עבודה" - כדי לא לשבור את האתר החי | Claude Code פותח branch בכל סשן |
| **Pull Request** | "בקשת מיזוג" - GitHub מציג מה השתנה | נפתח אוטומטית ב-GitHub |
| **Merge** | "אישור ופרסום" - מעביר שינויים ל-main | **אתה עושה ידנית** ב-GitHub |
| **main** | ה-branch הראשי - מה שרואים באתר החי | Vercel עוקב אחריו |

---

## מה אתה צריך לעשות בפועל

### שלב 1: לפתוח סשן עם Claude Code
פשוט מדברים. מסבירים מה רוצים. Claude Code עושה את העבודה.

### שלב 2: Claude Code דוחף שינויים
בסוף העבודה, Claude Code עושה commit + push אוטומטית ל-branch נפרד.
**אתה לא צריך לעשות כלום בשלב הזה.**

### שלב 3: מיזוג ל-main (אתה עושה)
1. נכנסים ל-GitHub → הריפו שלך
2. תראה הודעה צהובה: "claude/work-session-XXX had recent pushes"
3. לוחצים **"Compare & pull request"**
4. לוחצים **"Create pull request"**
5. לוחצים **"Merge pull request"**
6. לוחצים **"Confirm merge"**

**זהו. תוך דקה Vercel יבנה מחדש והאתר מתעדכן.**

### שלב 4: לבדוק שהכל עובד
נכנסים לאתר ובודקים שהשינויים נראים טוב.

---

## כללים חשובים

- **לא למזג לפני שבדקתם** - אם Claude Code שבר משהו, פשוט אל תמזגו
- **main = האתר החי** - כל מה שנמצא ב-main עולה לאוויר
- **Branch = מגרש ניסויים** - אפשר לשבור שם בלי לפגוע באתר
- **אם משהו השתבש אחרי מיזוג** - אפשר לעשות Revert ב-GitHub (כפתור ב-PR)

---

## שאלות נפוצות

**ש: מה ההבדל בין Commit ל-Push?**
Commit = שמירה מקומית על המחשב. Push = העלאה ל-GitHub.
בדרך כלל עושים את שניהם ביחד, ו-Claude Code מטפל בזה.

**ש: למה צריך Branch?**
כדי שאם משהו משתבש, האתר החי לא נפגע.
Branch זה כמו טיוטה - עובדים עליה ורק כשמרוצים, ממזגים ל-main.

**ש: מה קורה אם מיזגתי ומשהו נשבר?**
נכנסים ל-GitHub, פותחים את ה-PR שמיזגתם, ולוחצים "Revert".
זה יחזיר את האתר למצב הקודם.

**ש: למה Vercel? מה הוא עושה?**
Vercel הוא השרת שמריץ את האתר. הוא עוקב אחרי main ב-GitHub.
ברגע שמשתנה משהו ב-main, הוא בונה גרסה חדשה ומעלה אותה לאוויר.
אתה לא צריך לגעת ב-Vercel בכלל.

**ש: חזרתי אחרי הפסקה ארוכה, מה עושים?**
פשוט פותחים סשן חדש עם Claude Code ומתחילים לעבוד.
Claude Code יודע ליצור branch חדש ולדחוף שינויים.
אתה רק צריך למזג ב-GitHub כשמסיימים.
