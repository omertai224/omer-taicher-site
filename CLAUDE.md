# כללי ברזל לפרויקט

## גופן Rubik - לא לגעת!
הגופן Rubik חייב להיטען **תמיד**, בכל מצב, בלי תנאים.
- לא משנה כמה זמן לוקח לו להיטען
- לא משנה איזה מכשיר או חיבור אינטרנט
- לא משנה מה הציון ב-PageSpeed
- אסור להחליף אותו ב-`display:optional` או להתנות את הטעינה שלו
- Rubik הוא הגופן של האתר, תמיד נטען, נקודה.

## תהליכי עבודה - עדכון שוטף
תהליכי העבודה נמצאים בתיקייה `pages/workflow/` ומחולקים לקבצים קטנים:
- **`index.html`** — השלד (CSS, ניווט, JS שטוען כל טאב בנפרד)
- **`workflow.html`** — תהליך עבודה כללי
- **`posts.html`** — נוסחת-על לפוסטים
- **`interactive.html`** — נוסחת-על להדרכות
- **`payments.html`** — מוצרים ותשלומים
- **`sitemap.html`** — מבנה האתר
- **`calls.html`** — ניתוח שיחות לקוחות
- **כל פעם שלומדים משהו חדש** במהלך עבודה (טכניקה, מוקש, שיטה טובה) - לעדכן את הקובץ המתאים ב-`pages/workflow/`
- זה כולל: באגים שחוזרים, פתרונות שעובדים, שיטות עבודה חדשות, דברים שחשוב לזכור
- הקבצים האלה הם הזיכרון בין סשנים - מה שכתוב שם ישרת סשנים עתידיים
- לא לחכות לסוף - לעדכן כשמגלים

## דף הבית (index.html) - CSS אינליין!
דף הבית טוען את `main.css` **אסינכרונית** (`media="print" onload="this.media='all'"`).
- כל CSS לרכיבים דינמיים חדשים חייב להיכתב **בתוך ה-`<style>` האינליין** בראש הדף
- אם שמים CSS רק ב-main.css, הרכיב יופיע **בלי עיצוב** בטעינה הראשונית
- זה לא רלוונטי לדפים אחרים (interactive/index.html כותב CSS אינליין בכל מקרה)
