# הנחיות לפרויקט

## Git - דחיפות מרוכזות
- תמיד לצבור כמה שינויים יחד ולעשות commit + push אחד בסוף.
- לא לדחוף אחרי כל שינוי קטן. לחכות עד שכל המשימה מסתיימת.
- אם המשתמש מבקש כמה דברים, לעשות הכל ואז commit אחד.

## מבנה הפרויקט
- אתר עומר טייכר - static site על Vercel
- פאנל ניהול: `/admin/` - ממשק ניהול עם גישה ל-GitHub API
- כל push ל-GitHub מפעיל deployment ב-Vercel, לכן חשוב לצמצם pushים

## מי שולט על מה

### המשתמש (דרך פאנל ניהול) - שומר ב-localStorage, דוחף ידנית:
- **גלריה** - gallery.json (קטגוריות, סדר, העלאה/מחיקת תמונות)
- **בלוג** - blog/posts.json (כתיבה/עריכה/מחיקת פוסטים)
- **הדרכות אינטראקטיביות** - interactive/interactive.json (הוספה/עריכה/מחיקת הדרכות)
- **הנחיות (prompts)** - prompts.json (עדכון הנחיות AI)

### Claude (ישירות בקוד) - שומר ישירות ל-GitHub:
- **תוכן דף הבית** - content.json
- **תוכן אינטראקטיבי** - interactive/content.json
- **קוד, סגנון, לוגיקה** - כל קבצי JS/CSS/HTML
