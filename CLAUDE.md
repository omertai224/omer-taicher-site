### ניתוח תמלולים — כלל ברזל!
- **שני ריפוס במקביל:** `Calls` (פרטי — תמלולים ב-`inbox/` על main, catalog.json, analysis_log.json) + `omer-taicher-site` (רעיונות מזוקקים בתיקיית `ideas/`)
- **תמלולים על main:** כל התמלולים יושבים על בראנצ׳ `main` בריפו `omertai224/Calls`. Claude משתכפל מ-main ומושך הכל. תמלולים לא נמחקים.
- **analysis_log.json = מקור האמת** — קורא אותו ראשון כדי לדעת מה כבר נותח. תמלולים מתחת ל-200 מילים = קצר מדי, לדלג
- **תובנות מחולקות לקבצים קטנים** ב-`insights/categories/` (8 קבצים לפי נושא). **לא לקרוא את כולם בבת אחת!** קרא `insights/index.json` ואז רק את הקובץ הרלוונטי
- **שיטת ניתוח מוכחת — מגה-סבבים (עדכון מרץ 2026):**
  - **הגישה:** 5 אייג'נטים במקביל, כל אחד מנתח כמה שיחות. גודל הבאצ' לפי אורך הקבצים:
    - קבצים קצרים (עד 200 שורות): **10 קבצים לאייג'נט** = 50 בסבב
    - קבצים בינוניים (200-500 שורות): **5 קבצים לאייג'נט** = 25 בסבב
    - קבצים ארוכים (500+ שורות): לאייג'נט לקרוא רק 300 שורות ראשונות, ניתוח חלקי
    - קבצים ענקיים (1000+ שורות): אייג'נט בודד לקובץ בודד, בזהירות
  - **פורמט:** כל אייג'נט מחזיר JSON array עם אובייקטים: `file, client, lines, quality (1-5), quality_notes`
  - **שמירה:** אסוף תוצאות מכל האייג'נטים, הוסף ל-analysis_log.json בבת אחת (לולאת Python), commit + push
  - **דילוג בבאלק:** קבצים מתחת ל-200 מילים — לדלג על כולם בפעולה אחת, `status: "skipped"`
  - **אל תשאל את עומר, פשוט תמשיך!**
- **ביקורת אחרי כל גל גדול (חובה!):**
  1. כיסוי מלא — אין קבצים חסרים (inbox - analysis_log = 0)
  2. אין כפילויות ב-analysis_log
  3. אין דילוגים בטעות — קבצים 200+ מילים שסומנו skipped
  4. כל רשומה analyzed יש לה client, quality, quality_notes
  5. JSON תקין (json.dumps לא נופל)
  6. local = remote (git fetch + compare HEAD)
  7. working tree נקי (git status clean)
- **עבוד בחתיכות קטנות — דחוף אחרי כל סבב!** תמיד דחוף לשני הריפוזיטוריות
- **כל 5 סבבים (25 שיחות) — עדכן את האתר!** אל תצבור עשרות שיחות בלי לעדכן ideas/ באתר. כל 5 סבבים: בדוק מה חדש בתובנות, הוסף רעיונות חדשים לאתר, ודחוף. אם עומר מבקש לסדר באתר — עשה מיד
- **לא לגעת באתר!** ניתוח תמלולים הולך רק ל-"רעיונות להמשך" (תיקיית `ideas/`). לא משנים דפים, לא מוסיפים תוכן לאתר, לא נוגעים בקוד — עד שעומר נותן אישור מפורש

### סדר עדיפויות — מה שעושה כסף קודם!
1. **הדרכות אינטראקטיביות** — המקור המרכזי לכסף! 27-97 ש"ח, חטיפים. כל בעיה שחוזרת = רעיון להדרכה. **זה החלק המאתגר** — דורש בניית מוצר, לא רק כתיבה.
2. **מוצרים ושירותים** — חבילות, ריטיינר, שירותי פרימיום. עושים כסף ישירות.
3. **פוסטים לבלוג** — **עומר עושה בצ'יפס!** פוסטים קלים לו, לא צריך להתמקד בהם. הם מביאים תנועה ומובילים להדרכות.
4. **שיפורי מכירות, אתר, שיווק**
- **הכל חשוב!** הכל מערכת אחת שמזינה אחד את השני. הדרכות אינטראקטיביות הן הלב — כל השאר תומך, מוביל, ומזין אותן. שום דבר לא נזרק או מדולג.
- **עיקרון מנחה:** מה שיותר קל לעומר לייצר = קודם. כמות תוכן > שלמות. עדיף 10 הדרכות "טובות" מ-3 "מושלמות".

### דירוג איכות (1-5) לכל קובץ

### בעיית תמלולים פגומים
הרבה תמלולים סובלים מחזרות מאסיביות ("תודה", "כן", "זהו" חוזרות מאות פעמים). התוכן האמיתי הוא לרוב 10%-30% מהקובץ. לכן:
- קבצים עם 1000+ שורות לא בהכרח עשירים — יכולים להיות 90% רעש
- כשקוראים קובץ ארוך, להתחיל מ-300 שורות ראשונות ולהעריך
- אם התמלול פגום, לתת quality 1-2 ולהמשיך הלאה

### מבנה קבוע לכל הדרכה אינטראקטיבית
כל הדרכה חייבת לעקוב אחרי המבנה הזה:

| # | סוג | עיגול | תוכן |
|---|------|-------|-------|
| 1 | **פתיחה** | 🏠 home | שם ההדרכה, תיאור, "בואו נתחיל" |
| 2 | **סרטון ניווט** | ▶ play | "איך עובדים עם ההדרכה" (אותו סרטון בכל ההדרכות) |
| 3 | **הכירו את X** | 🎤/🔍 (לפי תוכנה) | placeholder/סרטון על התוכנה הספציפית |
| 4 | **הורדה** (אם צריך) | ⬇ download | "הורידו את X" — רק אם התוכנה דורשת הורדה |
| 5+ | **צעדים** | 1, 2, 3... | תמונה + בועת טקסט + מסגרת כתומה |
| ? | **שקפי מעבר** | 🖥 monitor | בין חלקים (הורדה→התקנה→שימוש) |
| אחרון | **סיום** | 💡 finish | "מה עכשיו?" — רעיונות מה לעשות עם התוצאה |

**סדר שקפים 2-3:** קודם ניווט (המשתמש לא מכיר את הפורמט), אח"כ תוכנה.

**אייקון שקף 3 לפי סוג תוכנה:**
- תמלול/אודיו → mic (מיקרופון)
- חיפוש → search (זכוכית מגדלת)
- כללי → monitor (מסך)

### פס ניווט תחתון — מפרט מדויק (מקור האמת!)
**זה המפרט הסופי והמדויק. כל הדרכה חדשה חייבת לעמוד בו בדיוק.**

#### מבנה הפס — 3 רכיבים קבועים:
```
[לוגו]  [════════ עיגולי ניווט ════════]  [◄ ►]
 שמאל        מתפרשים על כל הרוחב         ימין
```
1. **לוגו** — קבוע בשמאל למטה
2. **עיגולי ניווט** — גריד דינמי שתופס את כל הרוחב הפנוי
3. **חיצי ניווט** — קבועים בימין למטה

#### חוקי עיגולים — שתי שורות מרווחות:
- **עמודות = ceil(סה"כ_עיגולים / 2)** — תמיד מחלקים ל-2 שורות
- **אי-זוגי:** שורה עליונה ארוכה יותר ב-1, שורה תחתונה קצרה ב-1
- **זוגי:** שתי שורות שוות, אחד מתחת לשני בדיוק
- **כיוון: שמאל→ימין** (direction: ltr) — מספר 1 בשמאל
- **שקפים מיוחדים = אייקון SVG** (פתיחה, סרטון, הורדה, שימוש, סיום) — בלי מספר
- **צעדים רגילים = ממוספרים** מ-1 עד N רציף (רק שקפים עם תמונה+הוראה)
- **slideMap** מגדיר לכל הדרכה אילו שקפים מיוחדים (אינדקס 0-based → אייקון)
- **אייקונים זמינים:** home, play, download, monitor, search, finish
- **עיגולים מתפרשים על כל הרוחב** בין הלוגו לחיצים (1fr, לא 26px!)
- **צבע active:** צעדים = נייבי (#1a2540) עד המיקום הנוכחי, בהיר (#a8c5d6) אחרי. **מיוחדים = כתום כהה (#e8834e) עד המיקום, כתום בהיר (#f6a67e) אחרי**

```javascript
  let slides = document.getElementsByClassName("mySlides");
  let container = document.querySelector('.nav-dots');
  if (!container) return;
  var cols = Math.ceil(slides.length / 2);
  container.style.gridTemplateColumns = 'repeat(' + cols + ', 1fr)';
  var stepNum = 0;
  for (let i = 0; i < slides.length; i++) {
    let dot = document.createElement('button');
    dot.className = 'nav-dot';
    var special = slideMap[i];
    if (special) {
      // שקף מיוחד — אייקון SVG (בלי מספר!)
      dot.innerHTML = specialIcons[special.icon] || '';
      dot.title = special.title;
      dot.classList.add('nav-dot-icon');
    } else {
      // צעד רגיל — מספר רציף
      stepNum++;
      dot.textContent = stepNum.toString();
      dot.title = stepNum.toString();
    }
    (function(index) {
      dot.addEventListener('click', function() { showSlides(index + 1); });
    })(i);
    container.appendChild(dot);
  }
}
```

**כלל חשוב:** כל הדרכה חדשה — להתאים את `slideMap` לפי המבנה שלה! אינדקסים 0-based.

### זכוכית מגדלת (Magnifier) — חובה בכל הדרכה!
**פיצ'ר קריטי לנגישות מבוגרים.** כפתור מובנה שמאפשר הגדלת כל תוכן השקף — תמונות, בועות טקסט, ומסגרות כתומות.

#### למה זה כל כך חשוב:
- **הלקוחות של עומר הם מבוגרים** — קשה להם לראות פרטים קטנים בתמונות
- **מייתר חיתוך תמונות** — לא צריך יותר לחתוך תמונות שיהיו גדולות. תמונה מלאה + הגדלה = מושלם
- **שידרוג ברמות** — עומר אומר שזה "מיליון דולר" ו"משדרג את ההדרכה ברמות אחרות"

#### איך זה עובד:
1. **כפתור 80px** עם אייקון זכוכית מגדלת — ממורכז בחלק השחור שמשמאל לתמונה
2. **מירכוז אנכי** לפי החלק הנראה בלבד (מוריד 80px של פס ניווט מהחישוב)
3. **לחיצה** מפעילה מצב הגדלה — הכפתור נצבע כתום
4. **עדשה מרובעת 420×300** (פינות מעוגלות) עוקבת אחרי העכבר
5. **הגדלה ×2** — שכפול DOM של כל ה-image-center (לא background-image!), כך שגם טקסט וגם תמונות מוגדלים
6. **לחיצה נוספת** או **Escape** מכבה
7. **מופיע רק בשקפי צעדים** — לא במסך פתיחה/סיום/מעבר
8. **Esc hint** מופיע מתחת לכפתור כשהזכוכית פעילה — "ניתן ללחוץ Esc במקלדת לביטול"

#### מפרט טכני:

- **JS functions:** `initMagnifier()`, `toggleMagnifier()`, `positionEscHint()`, `updateMagnifierVisibility()`, `positionMagnifierBtn()`, `refreshMagnifierClone()`

#### מוקשים שנפתרו:

### שלב 9: הכנסת הדרכה מוכנה למאגר ולמכירה — צ'קליסט מלא

**Vibe הוא המודל המושלם.** כל הדרכה חדשה חייבת לעבוד בדיוק כמוהו — אותו זרימת קנייה, אותם מיילים, אותו WhatsApp, אותו מנגנון "מחובר כ:".

**כשהדרכה מוכנה (HTML + שקפים + ניווט עובד) — עושים את 5 הצעדים האלה:**

#### צעד 1: הוספה לקטלוג — `interactive/interactive.json`
```json
{
  "key": "everything",
  "title": "כותרת קצרה ומושכת",
  "desc": "תיאור של 1-2 משפטים",
  "steps": 31,
  "url": "./tutorials/Everything/index.html",
  "price": 47,
  "category": "כלים · חיפוש · Windows",
  "status": "active",
  "label": "חדש",
  "thumb": "teal",
  "includes": ["נקודה 1", "נקודה 2", "נקודה 3"]
}
```
- **`key`** = מזהה ייחודי (אותיות קטנות, בלי רווחים) — משמש בכל המערכת!
- **`status`**: `"active"` = למכירה, `"coming_soon"` = מוצג עם "בקרוב"
- **`thumb`**: צבע האייקון — `"navy"` / `"teal"` / `"purple"`
- **`includes`**: 3 נקודות שמופיעות בכרטיס המוצר
- **זה מקור האמת!** — דף הבית, פאנל ניהול, ו-checkout כולם קוראים מכאן

#### צעד 2: הוספה למערכת תשלום — `api/payplus.js`
שני מקומות לעדכן:

**א. TUTORIAL_URLS** (שורה ~71) — לאן הלקוח מופנה אחרי תשלום מוצלח:
```javascript
const TUTORIAL_URLS = {
  vibe: 'https://omertai.net/interactive/tutorials/Vibe/',
  everything: 'https://omertai.net/interactive/tutorials/Everything/',  // ← הוסף
  // ...
};
```

**ב. Fallback products** (שורה ~33) — גיבוי אם interactive.json לא נטען:
```javascript
return {
  vibe:       { name: '...', price: 47 },
  everything: { name: '...', price: 47 },  // ← הוסף
};
```

#### צעד 3: הוספה ל-webhook — `api/payplus-webhook.js`
**זה הקובץ הקריטי!** מכאן נשלחים המייל וה-WhatsApp אחרי תשלום.

```javascript
const PRODUCTS = {
  vibe:       { name: '...', url: 'https://omertai.net/interactive/tutorials/Vibe/' },
  everything: { name: '...', url: 'https://omertai.net/interactive/tutorials/Everything/' },  // ← הוסף
};
```

**חובה:**
- **`url`** — חייב להיות URL מלא (https://omertai.net/...) לא יחסי
- **`name`** — מופיע בנושא המייל ובהודעת WhatsApp

**מה קורה אחרי תשלום (אוטומטי):**
1. PayPlus שולח callback ל-`/api/payplus-webhook`
2. Webhook בודק `status_code === '000'` (אושר)
3. מוצא את המוצר לפי `more_info` (= productKey)
4. מוסיף `?u=שם_הלקוח` ל-URL (לינק אישי)
5. שולח **מייל** עם: שם המוצר + כפתור "עברו להדרכה"
6. שולח **WhatsApp** עם: שם המוצר + לינק אישי

#### צעד 4: מנגנון "מחובר כ:" — `index.html` של ההדרכה
**אין מסך לוגין! הלקוח נכנס ישירות להדרכה.** רק צריך 2 פונקציות לתצוגת שם:

```javascript
function getPersonalName() {
  var params = new URLSearchParams(window.location.search);
  var name = params.get('u');
  if (name) {
    localStorage.setItem('[key]_user_display', name);  // key = מזהה ההדרכה
    return name;
  }
  return localStorage.getItem('[key]_user_display') || '';
}

function showPersonalBadge() {
  var name = getPersonalName();
  if (!name) return;
  var badge = document.createElement('div');
  badge.id = 'user-badge';
  badge.innerHTML = '<svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="#5b8fa8" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" style="vertical-align:middle;margin-left:5px;"><path d="M20 21v-2a4 4 0 00-4-4H8a4 4 0 00-4 4v2"/><circle cx="12" cy="7" r="4"/></svg>' +
    '<span style="color:#ffffffaa;font-size:11px;">מחובר כ:</span> ' +
    '<span style="color:white;font-weight:700;font-size:12px;">' + name + '</span>';
  badge.style.cssText = 'position:fixed;top:12px;left:12px;z-index:9000;background:#0f1a2eee;border:1px solid #ffffff22;border-radius:20px;padding:6px 14px;font-family:Rubik,sans-serif;display:flex;align-items:center;gap:4px;backdrop-filter:blur(8px);box-shadow:0 4px 16px #00000044;';
  document.body.appendChild(badge);
}
```

**initApp() — פשוט! בדיקת מובייל ואז ישר להדרכה:**
```javascript
function initApp() {
  getPersonalName();
  var ua = navigator.userAgent || navigator.vendor || window.opera;
  if (/Android|.../.test(ua)) { document.getElementById('mobile-block').style.display = 'flex'; return; }
  showTutorial();
}
function showTutorial() { showPersonalBadge(); setNavBarColor(1); showSlides(1); }
```

**localStorage keys — חייבים להיות ייחודיים לכל הדרכה!**
- `[key]_user_display` — למשל `everything_user_display`

#### סיכום — מה הלקוח חווה אחרי הכל:
