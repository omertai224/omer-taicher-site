## תהליכי עבודה - הדבר הכי חשוב!
**בתחילת כל סשן חדש — קרא את תהליכי העבודה לפני כל דבר אחר.**
ככה אתה יודע בדיוק מה לעשות בלי שעומר צריך להסביר.

תהליכי העבודה נמצאים בתיקייה `pages/workflow/` ומחולקים לקבצים קטנים:
- **`workflow.css`** — עיצוב (כמעט אף פעם לא נוגעים)
- **`workflow.js`** — לוגיקת טעינת טאבים ותתי-לשוניות
- **`index.html`** — השלד (nav, tabs, footer)
- **`workflow.html`** — תהליך עבודה כללי, כללים, מוקשים ולקחים
- **`posts.html`** — נוסחת-על לפוסטים
- **`interactive.html`** — נוסחת-על להדרכות
- **`payments.html`** — מוצרים ותשלומים
- **`sitemap.html`** — מבנה האתר
- **`ideas.html`** — טאב "רעיונות להמשך" (שלד עם תתי-לשוניות)
- **`ideas/`** — תיקייה עם תתי-לשוניות: priorities, blog-ideas, interactive-ideas, products, sales, site, marketing
- **`calls.html`** — ניתוח שיחות לקוחות

### ניתוח תמלולים — כלל ברזל!
- **שני ריפוס במקביל:** `clients-calls` (פרטי — תמלולים, status_data.json, catalog.json, analysis_log.json) + `omer-taicher-site` (רעיונות מזוקקים בתיקיית `ideas/`)
- **status_data.json = מקור האמת** — קובץ JSON מובנה שהסקריפט מייצר. קורא אותו ראשון. תמלולים מתחת ל-200 מילים = קצר מדי, לדלג
- **לא לגעת באתר!** ניתוח תמלולים הולך רק ל-"רעיונות להמשך" (תיקיית `ideas/`). לא משנים דפים, לא מוסיפים תוכן לאתר, לא נוגעים בקוד — עד שעומר נותן אישור מפורש
- **כל רעיון חייב בשר:** לא משפט אחד — כל רעיון = כרטיסיית אקורדיון עם: "מה הרעיון", "למה יעבוד", "איך ליישם" (כולל איפה באתר, מה לשנות)
- מספיק פרקטי שעומר יכול להגיד "תעשה את זה" ולקבל תוצאה מוגמרת
- רעיונות הולכים לתת-הלשונית המתאימה: blog-ideas, interactive-ideas, products, sales, site, marketing
- **אחרי ניתוח — למחוק מ-inbox.** התמלול נמחק, הלוג שומר hash. אם עומר יעלה שוב בטעות — לדלג ולהודיע
- **שום מידע אישי של לקוחות לא עובר ל-omer-taicher-site** — רק רעיונות נקיים

### עדכון שוטף
- **כל פעם שלומדים משהו חדש** במהלך עבודה (טכניקה, מוקש, שיטה טובה) - לעדכן את הקובץ המתאים
- **בסוף כל סשן** - לבדוק אם יש משהו חדש שכדאי לתעד בתהליכי העבודה
- זה כולל: באגים שחוזרים, פתרונות שעובדים, שיטות עבודה חדשות
- הקבצים האלה הם הזיכרון בין סשנים - מה שכתוב שם ישרת סשנים עתידיים

## דף הבית (index.html) - CSS אינליין!
דף הבית טוען את `main.css` **אסינכרונית** (`media="print" onload="this.media='all'"`).
- כל CSS לרכיבים דינמיים חדשים חייב להיכתב **בתוך ה-`<style>` האינליין** בראש הדף
- אם שמים CSS רק ב-main.css, הרכיב יופיע **בלי עיצוב** בטעינה הראשונית
- זה לא רלוונטי לדפים אחרים (interactive/index.html כותב CSS אינליין בכל מקרה)
