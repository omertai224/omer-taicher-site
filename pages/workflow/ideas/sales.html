הרעיון בקצרה

3 הדרכות (סיסמאות + גיבוי + אנטי-וירוס) ב-119₪ במקום 141₪. שם שמדבר לרגש — "שקט נפשי" = אני מוגן.

למה זה יעבוד
  • הקוד כבר תומך בפרמטר cart — צריך רק כרטיס בדף
  • באנדל מעלה ערך עסקה ממוצע ב-70%+
  • הנחה קטנה (22₪) אבל מרגישה כמו "מציאה"
  • 3 מוצרים שמשלימים אחד את השני = הגיוני לקנות ביחד
איך ליישם
  • הוספת כרטיס "חבילת שקט נפשי" בדף ההדרכות — עיצוב בולט, תגית "הכי פופולרי"
  • הגדרת פרמטר cart עם 3 ההדרכות
  • בדיקה שה-checkout תומך בכמה מוצרים ביחד

הרעיון בקצרה

צ'קבוקס בדף התשלום: "רוצה שעת תמיכה מרחוק? +79₪". רשת ביטחון שנותנת ביטחון = יותר מכירות.

למה זה יעבוד
  • הקהל מפחד "מה אם לא אצליח לבד?" — התמיכה מורידה את החסם
  • 79₪ = זול מטכנאי. אבל רוב האנשים לא ישתמשו — רווח נקי
  • אפסייל ברגע התשלום = הרגע הכי טוב למכירה נוספת
איך ליישם
  • צ'קבוקס מעוצב בדף ה-checkout: "הוסף שעת תמיכה מרחוק (+79₪)"
  • אם נבחר — הוספת מוצר נוסף לעסקה
  • מייל אוטומטי עם לינק לתיאום שעת תמיכה

הרעיון בקצרה

"נהנית מההדרכה? הנה עוד אחת ב-20% הנחה." מייל אוטומטי ש-Brevo שולח 3 ימים אחרי רכישה.

למה זה יעבוד
  • Brevo כבר מחובר ותומך ב-automations — אפס עלות
  • 3 ימים = מספיק זמן לסיים את ההדרכה, מוקדם מספיק שהחוויה עדיין חיובית
  • 20% הנחה על המוצר הבא = תמריץ חזק למכירה חוזרת
  • מייל אחד = הכנסה פסיבית לנצח
איך ליישם
  • Brevo: יצירת automation trigger — 3 ימים אחרי רכישה
  • עיצוב מייל פשוט: "תודה שרכשת [שם הדרכה]. הנה עוד אחת שתתאים לך."
  • קוד הנחה ייחודי שפג תוקף אחרי 7 ימים = דחיפות

הרעיון בקצרה

שדה "קוד הנחה" בדף התשלום. תשתית שמאפשרת הנחות ללקוחות חוזרים, למשתתפי הרצאה, לחברי קבוצת פייסבוק.

למה זה יעבוד
  • מאפשר פרומושנים ממוקדים — "הרצאה שלי בשבוע הבא? הנה קוד 15% הנחה"
  • לקוחות חוזרים מרגישים VIP — מגדיל נאמנות
  • מאפשר למדוד מאיפה מגיעות מכירות (כל קוד = מקור שיווקי)
איך ליישם
  • שדה קלט בדף ה-checkout: "יש לך קוד הנחה?"
  • JSON פשוט עם קודים + אחוזי הנחה + תאריך תפוגה
  • בדיקת תקינות בצד שרת (Vercel function)
