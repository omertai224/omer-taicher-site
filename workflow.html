כללים חשובים
main = האתר החי. כל מה שנמצא ב-main עולה לאוויר אוטומטית.
Branch = מגרש ניסויים. אפשר לשבור שם בלי לפגוע באתר.
לא למזג לפני שבדקתם. אם Claude Code שבר משהו - פשוט אל תמזגו.
משהו נשבר אחרי מיזוג? נכנסים ל-GitHub, פותחים את ה-PR ולוחצים "Revert".
מוקשים ולקחים
דף הבית: CSS חייב להיות אינליין. main.css נטען אסינכרונית. כל CSS לרכיב חדש חייב להיכתב בתוך ה-<style> בראש הדף, אחרת הרכיב יופיע בלי עיצוב.
popup שנצמד לכרטיס = בעיות. אם צריך תוכן שקופץ (דרישות מקדימות, מידע נוסף), להשתמש במודל מרכזי (overlay + modal) ולא ב-popup שמנסה להיצמד לכרטיס.
interactive.json = מקור אמת למוצרים. כל מידע על הדרכה (מחיר, תיאור, דרישות מקדימות) נמצא שם. שני הדפים (בית + אינטראקטיבי) קוראים ממנו.
דרישות מקדימות הן לכל הדרכה בנפרד. שדה prereqs ב-interactive.json. לא גנרי לכל ההדרכות - כל הדרכה עם הדרישות שלה.
שאלות נפוצות
מה ההבדל בין Commit ל-Push?
Commit = שמירה על המחשב. Push = העלאה ל-GitHub.
כמו לכתוב טיוטה (commit) ואז לשלוח אותה במייל (push).
Claude Code עושה את שניהם ביחד - אתה לא צריך לדאוג לזה.
למה צריך Branch?
כדי שאם משהו ישתבש, האתר החי לא ייפגע.
Branch זה כמו טיוטה - עובדים עליה, ורק כשמרוצים ממזגים ל-main.
מה עושה Vercel?
Vercel הוא השרת שמריץ את האתר שלך.
הוא עוקב אחרי main ב-GitHub. ברגע שמשתנה משהו שם - הוא בונה גרסה חדשה ומעלה לאוויר.
אתה לא צריך לגעת ב-Vercel בכלל.
חזרתי אחרי הפסקה ארוכה, מה עושים?
פשוט פותחים סשן חדש עם Claude Code ומתחילים לעבוד.
Claude Code יודע ליצור branch חדש ולדחוף שינויים.
אתה רק צריך למזג ב-GitHub כשמסיימים.
נוסחת-על: כתיבת פוסטים לבלוג

המטרה: לייצר פוסט חדש לבלוג בקובץ JSON, בסגנון אחיד ועקבי, עם מינימום עבודה ידנית.

מבנה קובץ JSON
id
מזהה ייחודי
שם הקובץ וחלק מה-URL. באנגלית, עם מקפים. לדוגמה: ai-prompt-clear-goal
title
כותרת הפוסט
עטוף בתגיות <p> כמו בפוסטים הקיימים. שורה-שורה.
excerpt
תקציר קצר
עטוף בתגיות <p>. מוצג בכרטיס הפוסט בדף הבלוג.
body
גוף הפוסט
HTML עם h2 + SVG, post-steps, blockquote, post-insight — לפי מה שמתאים לתוכן.
date
תאריך פרסום
פורמט: YYYY-MM-DD. ברירת מחדל: היום.
image
תמונה ראשית
תמיד בפורמט: https://media.omertai.net/{id}.webp
image_alt
תיאור ויזואלי
תיאור מפורט שישמש ליצירת התמונה. לא סתם alt text.
seo_title / seo_desc
מטא לקידום
כותרת ותיאור SEO. הכותרת עד 60 תווים, התיאור עד 155.
תהליך העבודה
1
כותבים פוסט גולמי — טקסט חופשי, בלי HTML, בלי עיצוב. רק התוכן.
2
מוסרים לקלוד עם ההנחיה: "יש לי פוסט גולמי. תסתכל על הפוסטים הקיימים ב-blog/posts/ ותבנה JSON חדש באותו מבנה ועיצוב."
3
קלוד בודק את הפוסטים הקיימים, לומד את המבנה, ובונה קובץ JSON חדש עם כל השדות.
4
קלוד מעדכן את posts-index.json ומוסיף את הפוסט במיקום הנכון לפי תאריך.
5
Commit + Push — קלוד שומר ודוחף. אתה ממזג ב-GitHub.
כללים חשובים
כל השדות חובה: id, title, excerpt, body, date, image, image_alt, seo_title, seo_desc
title ו-excerpt עטופים ב-<p> — כמו בפוסטים הקיימים בדיוק.
body כולל: h2 עם SVG, post-steps, blockquote, post-insight — לפי מה שמתאים.
image תמיד: https://media.omertai.net/{id}.webp
image_alt הוא תיאור ויזואלי מפורט — ישמש ליצירת התמונה, לא סתם נגישות.

שיפורי מכירות
באנדל "חבילת שקט נפשי" - 3 הדרכות ב-119 שקל במקום 141. הקוד כבר תומך בפרמטר cart. צריך רק להוסיף כרטיס חבילה בדף ההדרכות.
"תוספת: שעת תמיכה" בצ'קאאוט - צ'קבוקס בדף התשלום: "רוצה שעת תמיכה מרחוק? +79 שקל". רשת ביטחון = יותר מכירות.
מייל מעקב 3 ימים אחרי רכישה - "נהנית מההדרכה? הנה עוד אחת ב-20% הנחה." Brevo כבר מחובר ותומך ב-automations.
מערכת קופונים - שדה "קוד הנחה" בצ'קאאוט. מאפשר הנחות ללקוחות חוזרים, למשתתפי הרצאה, לחברי קבוצת פייסבוק.
שיפורי אתר
CTA בסוף כל פוסט בבלוג - כל פוסט צריך להסתיים בקישור להדרכה רלוונטית. "רוצים לסדר את כל הסיסמאות? יש הדרכה בדיוק על זה." שינוי קטן, אפקט מיידי.
איסוף מיילים בבלוג - טופס "רוצים טיפ כזה כל שבוע?" בסוף כל פוסט. Brevo כבר מחובר. כל מבקר בבלוג שלא משאיר מייל = ליד שנעלם.
עמוד "תודה" אחרי רכישה - אחרי תשלום, הלקוח מועבר ישירות להדרכה. להוסיף עמוד ביניים עם הזמנה להרצאה, קישור לוואטסאפ, והצעה להדרכה נוספת. הרגע אחרי רכישה = הסיכוי הגבוה ביותר לעוד רכישה.
באנר הרצאה חינמית בדף הבית - ההרצאה מוסתרת בתפריט. להוסיף באנר בולט עם ספירה לאחור: "ההרצאה הקרובה בעוד X ימים". הרצאות חינמיות = משפך מכירות מוכח.
שיווק ותוכן
"שאלות שאני שומע כל שבוע" - כל שאלה שחוזרת בקבוצת הפייסבוק = פוסט בבלוג + קישור להדרכה. תוכן שלא צריך להמציא. Claude Code יכול לעזור ליצור אותם מהר.
עמוד "הרצאות לארגונים" - אין דף שמציע הרצאות לחברות וארגונים. עמוד אחד עם 3-4 נושאים + טופס פנייה = הזמנות ב-2,000-5,000 שקל להרצאה.
