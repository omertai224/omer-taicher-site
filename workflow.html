פתיחת סשן עבודה חדש
אחרי שמחקתם את הבראנץ' הישן ופותחים סשן חדש:
1
כותבים לקלוד: "עבודה" — קלוד יוצר בראנץ' חדש (סשן עבודה)
2
כותבים לקלוד: "דחוף את העץ" — קלוד דוחף את הבראנץ' ל-GitHub
3
נכנסים ל-GitHub → מעתיקים את קישור ה-PR של הבראנץ' החדש → מעדכנים את המועדף "מיזוג" בדפדפן עם ה-URL החדש
4
נכנסים ל-Settings בפרויקט ב-GitHub → Pages → משנים את הבראנץ' של GitHub Pages לבראנץ' החדש
5
רק עכשיו המועדף "Preview" יעבוד ויציג את השינויים מהבראנץ' החדש דרך GitHub Pages
מילון מושגים
Commit
שמירת נקודת ציון
כמו "שמור" בוורד. Claude Code עושה את זה בשבילך.
Push
העלאה ל-GitHub
לוקח את מה ששמרת מקומית ומעלה לענן. Claude Code עושה את זה בשבילך.
Branch
עותק עבודה
כמו טיוטה. עובדים עליה בלי לפגוע באתר החי. Claude Code פותח אחד בכל סשן.
Pull Request
בקשת מיזוג
GitHub מציג מה בדיוק השתנה ומבקש אישור לפני שזה עובר ל-main.
Merge
מיזוג = פרסום
מעביר את השינויים מהטיוטה ל-main. זה הרגע שהאתר מתעדכן. אתה עושה את זה.
main
הענף הראשי = האתר החי
מה שנמצא ב-main הוא מה שמוצג באתר omertai.net. Vercel עוקב אחריו.

שאלות נפוצות
מה ההבדל בין Commit ל-Push?
Commit = שמירה על המחשב. Push = העלאה ל-GitHub.
כמו לכתוב טיוטה (commit) ואז לשלוח אותה במייל (push).
Claude Code עושה את שניהם ביחד - אתה לא צריך לדאוג לזה.
למה צריך Branch?
כדי שאם משהו ישתבש, האתר החי לא ייפגע.
Branch זה כמו טיוטה - עובדים עליה, ורק כשמרוצים ממזגים ל-main.
מה עושה Vercel?
Vercel הוא השרת שמריץ את האתר שלך.
הוא עוקב אחרי main ב-GitHub. ברגע שמשתנה משהו שם - הוא בונה גרסה חדשה ומעלה לאוויר.
אתה לא צריך לגעת ב-Vercel בכלל.
חזרתי אחרי הפסקה ארוכה, מה עושים?
פשוט פותחים סשן חדש עם Claude Code ומתחילים לעבוד.
Claude Code יודע ליצור branch חדש ולדחוף שינויים.
אתה רק צריך למזג ב-GitHub כשמסיימים.
